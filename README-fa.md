# 🦢 گوس‌من (Gooseman)

گوس‌من یک داشبورد وب سبک برای مدیریت و مانیتور کردن کلاینت درحال اجرای GooseRelayVPN روی سیستم محلی یا سرور LAN است.

این پروژه یک پنل کنترل ساده برای استارت و استاپ کردن کلاینت، مشاهده لاگ‌های لحظه‌ای، دنبال کردن آمار مصرف، و ویرایش تنظیمات پراکسی SOCKS از طریق مرورگر فراهم می‌کند.

## ویژگی‌ها

- داشبوردی زیبا، مدرن و چشم‌نواز
- پشتیبانی از باینری‌های ویندوز و لینوکس
- امکان استارت و استاپ کردن کلاینت GooseRelayVPN از طریق رابط وب
- نمایش زنده لاگ‌ها با بروزرسانی خودکار
- نمایش لحظه‌ای آمار سشن و ترافیک
- دنبال کردن میزان کوتا (Quota)
- احراز هویت داشبورد بر پایه پسورد پراکسی SOCKS5
- ویرایشگر تنظیمات SOCKS5 (هاست، پورت، نام کاربری و رمز عبور اختیاری)
- طراحی واکنش‌گرا برای دسکتاپ و موبایل
- بک‌اند سبک مبتنی بر FastAPI بدون وابستگی خارجی به‌جز پکیج‌های پایتون
- نصب و راه‌اندازی بسیار آسان

---

## در نظر داشته باشید

- این پروژه از CDNهای tailwindcss و jsdelivr استفاده می‌کند که در حال حاضر همچنان در ایران فیلتر هستند. سرور بدون لود شدن آن‌ها هم به‌خوبی اجرا می‌شود، اما ظاهر زیبا، استایل‌ها و همچنین نمودارها نمایش داده نخواهند شد. البته به احتمال زیاد، پس از یک‌بار باز کردن سایت، فایل‌های استایل داخل مرورگر شما کش می‌شوند.

---

## معماری پروژه

گوس‌من از دو بخش اصلی تشکیل شده است:

- **بک‌اند (FastAPI)**
  - اجرای و مدیریت پروسه goose-client
  - خواندن و پردازش لاگ‌ها از خروجی استاندارد (stdout)
  - دنبال کردن آمار اجرا و میزان مصرف کوتا
  - ارائه یک API ساده HTTP برای داشبورد

- **فرانت‌اند (Single-page HTML)**
  - ساخته‌شده با TailwindCSS از طریق CDN
  - ارتباط با بک‌اند از طریق JavaScript Fetch API
  - نمایش زنده لاگ‌ها، آمار و کنترل‌ها

## پیش‌نیازها

- پایتون 3.9 یا بالاتر
- FastAPI
- Uvicorn
- باینری GooseRelayVPN 1.6 (`goose-client(.exe)`) در ریشه پروژه
- فایل `client_config.json` در ریشه پروژه

---

## نصب

1. کلون کردن ریپازیتوری:

```bash
git clone https://github.com/Aydiniyom/Gooseman.git
cd Gooseman
```

2. نصب وابستگی‌ها:

```bash
pip install -r requirements.txt
```

3. فایل‌های `client_config.json` و `goose-client(.exe)` خود را داخل پوشه اصلی پروژه (root directory) قرار دهید.

---

## اجرا

داشبورد را با دستور زیر اجرا کنید:

```bash
uvicorn main:app --host 0.0.0.0 --port 5000
```

سپس می‌توانید از طریق سیستم میزبان با آدرس `http://localhost:5000` یا از طریق دستگاه‌های دیگر داخل شبکه با آدرس `http://<host-machine-ip>:5000` به آن دسترسی داشته باشید.

> اگر پورت `5000` قابل bind شدن نبود، کافی است یک پورت تصادفی دیگر انتخاب کنید که از قبل اشغال نشده باشد.
